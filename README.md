🍺 第一屆 IT 居酒屋盃：人腦伺服器同步率大賽

(The 1st IT Izakaya Cup: Human Server Synchronization Protocol)

🎯 任務目標 (Mission Objective)

在這個充滿酒精與美食的夜晚，我們不追求標準答案，我們追求的是 「答案一致」！
本活動將測試各桌伺服器（小組）內的通訊協定是否同步。能不能拿到高分，全看你們與隔壁同事的「腦波頻率」是否在同一個網段上！

🕹️ 運算流程 (Process)

Request (接收請求)：
主持人宣讀題目。題目沒有標準答案，請依照直覺作答。

Processing (獨立運算)：
每個人在白紙/白板上寫下答案。

⚠️ 警告： 運算期間開啟防火牆 (Firewall)，禁止交頭接耳、禁止偷看隔壁程式碼。

Deploy (同步發佈)：
主持人倒數喊 「Deploy！」 時，全桌同時亮出答案。

Logging (結算積分)：
依照下方計分表 (Algorithm) 統計該回合得分。

📊 計分規則 (Scoring Algorithm)

為了確保不同架構的伺服器（3人桌與4人桌）公平競爭，本系統已導入 期望值平衡修正 (EV Balance Patch)。

🔹 4 人伺服器 (4-Node Cluster)

|

| 同步狀況 (Pattern) | IT 術語判定 | 得分 (Score) |
| 4 人答案相同 | Sync Success (完美同步) | 🏆 12 分 |
| 3 人答案相同 | Minor Bug (輕微異常) | 6 分 |
| 2 對 2 答案相同 | Split Brain (腦裂現象) | 6 分 |
| 2 人答案相同 | Partial Sync (部分同步) | 2 分 |
| 4 人皆不同 | System Crash (系統崩潰) | 0 分 |

🔸 3 人伺服器 (3-Node Cluster)

| 3 人答案相同 | Sync Success (完美同步) | 🏆 10 分 |
| 2 人答案相同 | Majority Sync (多數決同步) | 5 分 |
| 3 人皆不同 | System Crash (系統崩潰) | 0 分 |

🎤 主持人開場講稿 (Host Script)

「各位 IT 部門的夥伴們，大家晚安！

平常我們都在管 Server、解 Bug、或是跟 User 搏鬥。今天我們不談工作，在居酒屋我們要來測試一下各位 『人腦伺服器』 的相容性！

等一下我會出題，題目都沒有標準答案。請大家用你們的 『直覺』 寫下答案。
重點來了：不要寫你覺得『對』的答案，要寫你覺得『隔壁同事』會寫的答案！

我們的計分系統已經經過精密的 『期望值演算法』 平衡過，所以不管你們這桌是 4 個人還是 3 個人，獲勝機率都是公平的。

現在，請拿起你們的白紙和筆，確認連線狀態，比賽開始！」

📝 備註 (Notes)

0 分懲罰 (Optional)： 若該局發生「系統崩潰 (System Crash)」，建議該桌全員罰喝一口茶/酒。

爭議判定： 若答案意思相近但不完全相同（例如：「寫程式」與「Coding」），由主持人判定是否視為同步 (Compatible)。
